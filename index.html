Android常用命令之adb

工作中经常需要用到一些adb命令，在此整理记录下来。

1、基本命令

adb devices 查看已连接设备
adb -s 指定连接设备
adb root 获取root权限
adb start-server/kill-server 启动/关闭adb服务
adb connect 通过IP地址连接设备

2、应用管理

adb shell pm list packages [] 查看应用列表

-d disable应用
-e enabled应用
-s 系统应用
-3 第三方应用

adb install 安装应用
-r 覆盖安装(INSTALL_FAILED_ALREADY_EXISTS的解决办法，或者先adb install)

博客怎么写，我来立flag

发表于 2017-11-15

毕业前后，发生了很多事情。写博客这个习惯不知怎的也渐渐放下了。之前在CSDN和简书都有一些学习的总结，不过那时都还在上大学。
写博客确实是一种很有用的学习方式，有些人可能会觉得是浪费时间，其实不然，能把一个东西讲清楚，必须先完全理解透，写的过程也就是在思考的过程，可以对某个疑难点再进行反复斟酌，可以对整体流程和内容再加深印象。再者，总结下来的博客可以方便以后查找，日积月累，这些内容输出会给你满满的收获感。
但是有些人可能觉得没东西输出，不知道怎么写。上学时候我写博客也是这种感觉，很多时候只是理论和概念的摘抄，却很少有自己的理解。其实，我觉得有两种办法解决这个问题。一是基于工作上遇到的问题，如果有自己更好的解决方法或者问题解决过程有学到新东西，都可以先记录到个人笔记，然后等有空的时候再把这些零碎的知识点整理写出来。二是可以找某一个专题点去学习和研究，比如内存优化等，通过网络或者书籍弄清楚这方面的内容，过程中肯定有自己的见解，及时记录后面再总结出来。
其实上面这些我随便说的==，都不重要，看自己个人学习方法吧。最最最重要的还是坚持。我也很难坚持做好一件事情，很多时候要强迫自己，很不舒服。但是我也清楚一件事，最安逸舒适的时候也就是在原地踏步的时候，前进总需要付出辛苦的汗水。感到困难就是因为遇到了难题或者在解决新问题，此时你正在进步，而熟练的问题是不会给你这种有压力的感觉的。
好吧，以后尽量每星期一篇，把学到的东西总结在这里，这样就慢慢积累起来了。最后就是少玩农药，多看书啦。
